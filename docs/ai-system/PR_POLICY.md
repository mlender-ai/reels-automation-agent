# PR Policy

## Default Position

Auto PR is allowed only for small, low-risk changes.

Automatic merge is disabled by default.

Every AI-generated PR must be reviewed by a human before merge.

## Allowed Auto PR Scope

- README or docs clarification
- prompt wording fixes
- import cleanup
- dead code removal
- small null checks or guard clauses
- loading, empty, and error state fixes
- label or text corrections
- small UI bugs
- low-risk config cleanup
- narrow tests or validation improvements

## Human-Only Scope

- architecture rewrites
- large refactors
- destructive DB changes
- auth system changes
- secret handling changes
- real platform upload integrations
- broad workflow rewrites

## Size Limits

- maximum changed files: 5
- maximum total diff lines: 220
- default bias: docs, UI, and small quality fixes

If a patch exceeds those limits, it must stay report-only.

## Label Policy

Draft PRs generated by automation should be prepared to carry:

- `ai-generated`
- `needs-human-review`

## Merge Policy

- auto merge is off by default
- CI failure blocks manual merge until fixed
- human approval is always required
