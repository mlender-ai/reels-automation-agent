# Handoff Template

Use this template when handing the repository to a new session, developer, or AI agent.

```md
# Handoff

## Current State
- What is working now?
- What is partially working?
- What is blocked?

## Recent Decisions
- Decision:
- Reason:
- Files affected:

## Next Work
- Immediate next task:
- Why it matters:
- Expected validation:

## Risks Or Unknowns
- Risk:
- Impact:
- Open question:

## Related Files
- /absolute/or/repo/path/to/file
- /absolute/or/repo/path/to/file

## Useful Commands
- backend:
- frontend:
- automation:
```

Keep handoffs concrete. The goal is to reduce rediscovery time for whoever picks up the repo next.
