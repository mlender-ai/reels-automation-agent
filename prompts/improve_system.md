# improve_system

Upgrade the repository from demo MVP to a stable local product that can keep evolving.

## Objective

Audit the current codebase and improve the areas most likely to break real usage.

## Focus Areas

- API and frontend integration mismatches
- transcript and clip generation reliability
- export and subtitle quality
- publish queue structure
- empty, loading, and error states
- documentation accuracy

## Guardrails

- do not rewrite the whole project
- preserve working flows
- replace weak placeholders with working logic where needed
- keep abstraction light and practical

## Expected Outcome

- fewer broken states
- better UI guidance
- cleaner service boundaries
- more reliable export and publish staging
- docs that match reality
